# 🎬 Демонстрация системы заданий RezoFlow

## Автоматическая демонстрация

Для быстрой демонстрации работы системы заданий по классам:

### Вариант 1: Два окна браузера

**Окно 1 - Ученик (8А класс):**
```typescript
// В App.tsx установите:
const [isAuthenticated, setIsAuthenticated] = useState(true);
const [userName, setUserName] = useState('Алексей');
const [userRole, setUserRole] = useState<'student' | 'teacher' | 'parent' | null>('student');
const [studentClass, setStudentClass] = useState<string>('8А');
```

**Окно 2 - Учитель:**
```typescript
// В App.tsx установите:
const [isAuthenticated, setIsAuthenticated] = useState(true);
const [userName, setUserName] = useState('Анна Викторовна');
const [userRole, setUserRole] = useState<'student' | 'teacher' | 'parent' | null>('teacher');
```

### Процесс демонстрации

1. **Откройте оба окна рядом**
   - Слева: дашборд ученика 8А
   - Справа: дашборд учителя

2. **В окне учителя:**
   - Нажмите "Создать задание"
   - Заполните форму:
     - Название: "Решить квадратные уравнения"
     - Предмет: "Математика"
     - Класс: **8А**
     - Дедлайн: завтра
     - Описание: "Учебник стр. 45, задачи №1-10"
   - Нажмите "Создать задание"

3. **В окне ученика (МГНОВЕННО):**
   - 🔔 Появится уведомление (красный кружок на колокольчике)
   - 📝 Задание автоматически добавится в список
   - ✨ Можно кликнуть на уведомление для перехода к заданию

4. **Продемонстрируйте фильтрацию:**
   - В окне учителя создайте задание для **7Б** класса
   - Ученик 8А НЕ получит это задание
   - Его уведомления останутся прежними

## Демо-сценарий для презентации

### Сценарий 1: "Живая доставка заданий"

**Установка:**
- Ученик 8А класса сидит на уроке
- Учитель дает домашнее задание

**Действие:**
1. Учитель открывает RezoFlow на проекторе
2. Создает задание для 8А класса
3. Ученики мгновенно видят уведомление на своих устройствах

**Эффект:** "Вау, это сразу пришло!"

---

### Сценарий 2: "Умная фильтрация"

**Установка:**
- 3 окна браузера с разными классами:
  - Ученик 7А
  - Ученик 8А
  - Ученик 9Б

**Действие:**
1. Учитель создает задание для 8А
2. Только окно ученика 8А получает уведомление
3. Остальные окна остаются без изменений

**Эффект:** "Каждый видит только свои задания!"

---

### Сценарий 3: "Полный workflow"

**Участники:**
- Учитель
- Ученик 8А

**Шаги:**
1. **Создание задания** (учитель)
   - Форма с AI-помощником
   - Выбор класса из списка
   - Добавление описания

2. **Получение** (ученик)
   - Уведомление в реальном времени
   - Звуковой сигнал (если включено)
   - Задание в списке

3. **Выполнение** (ученик)
   - Просмотр деталей
   - Загрузка скриншота
   - Отметка как выполненное

4. **Проверка** (учитель)
   - Вкладка "Проверка работ"
   - Просмотр скриншота
   - Принятие/отклонение

## Типичные вопросы во время демо

**Q: Можно ли отправить задание сразу нескольким классам?**
A: В текущей версии нет, но это в планах. Сейчас нужно создать отдельное задание для каждого класса.

**Q: Как ученик узнает, что пришло задание?**
A: Три способа:
1. Браузерное уведомление (если разрешено)
2. Уведомление в приложении (колокольчик)
3. Звуковой сигнал (опционально)

**Q: Может ли ученик увидеть задания других классов?**
A: Нет, система фильтрует задания строго по классам.

**Q: Что если ученик выбрал неправильный класс при регистрации?**
A: В текущей версии нужна новая регистрация. Функция смены класса в разработке.
