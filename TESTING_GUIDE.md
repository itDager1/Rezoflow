# 🧪 Руководство по тестированию системы заданий

## Быстрый тест функционала

### Шаг 1: Тестирование регистрации ученика

1. Откройте приложение
2. В `App.tsx` измените начальное состояние:
   ```typescript
   const [isAuthenticated, setIsAuthenticated] = useState(false); // было true
   const [userRole, setUserRole] = useState<'student' | 'teacher' | 'parent' | null>(null); // было 'student'
   ```

3. Зарегистрируйтесь как ученик:
   - Выберите роль "Школьник"
   - Появятся поля выбора класса
   - Выберите класс, например: **8А**
   - Заполните остальные данные

4. Проверьте:
   - ✅ После регистрации вы попали в дашборд ученика
   - ✅ Под логотипом RezoFlow видна плашка "Класс: 8А"
   - ✅ В профиле указано "Ученик 8А класса"

### Шаг 2: Тестирование создания задания учителем

1. Откройте новое окно браузера (или инкогнито)
2. В `App.tsx` установите:
   ```typescript
   const [userRole, setUserRole] = useState<'student' | 'teacher' | 'parent' | null>('teacher');
   ```

3. В дашборде учителя:
   - Нажмите "Создать задание"
   - Увидите информационный блок о системе заданий
   - Заполните форму:
     - Название: "Решить уравнения"
     - Предмет: "Математика"
     - **Класс: 8А** (из выпадающего списка)
     - Дедлайн: завтрашняя дата
     - Описание: "Страница 45, №1-10"

4. Нажмите "Создать задание"

### Шаг 3: Проверка получения задания учеником

Вернитесь в окно с учеником (8А класс):

1. Проверьте уведомления (колокольчик):
   - ✅ Должно появиться новое уведомление
   - ✅ "Новое задание от учителя"
   - ✅ Название и предмет задания

2. Проверьте список задач:
   - ✅ Задание "Решить уравнения" появилось в списке
   - ✅ Указан предмет "Математика"
   - ✅ Есть дедлайн

3. Нажмите на уведомление:
   - ✅ Страница прокручивается к заданию
   - ✅ Задание подсвечивается

### Шаг 4: Тест фильтрации по классам

Создайте еще одного ученика (другой класс):

1. Откройте третье окно браузера
2. Зарегистрируйте ученика класса **7Б**
3. Вернитесь к учителю
4. Создайте задание для **8А** класса

Проверьте:
- ✅ Ученик 8А получил задание
- ✅ Ученик 7Б НЕ получил задание
- ✅ У ученика 7Б в уведомлениях пусто

### Шаг 5: Тест с несколькими классами

В окне учителя:

1. Создайте задание для **7Б**:
   - Название: "Контрольная работа"
   - Предмет: "Физика"
   - Класс: **7Б**

2. Создайте задание для **8А**:
   - Название: "Домашняя работа"
   - Предмет: "Химия"
   - Класс: **8А**

Проверьте:
- ✅ Ученик 7Б видит только "Контрольная работа"
- ✅ Ученик 8А видит только "Домашняя работа"
- ✅ Каждый получил уведомление только о своём задании

## Проверка Edge Cases

### Кейс 1: Без указания класса

1. Попробуйте создать задание без выбора класса
2. ✅ Форма не должна отправиться (поле обязательное)

### Кейс 2: Несколько заданий подряд

1. Создайте 3 задания для одного класса подряд
2. ✅ Все 3 задания должны появиться у ученика
3. ✅ 3 уведомления в списке

### Кейс 3: Смена класса в профиле

1. Пока функционал не реализован
2. Класс определяется при регистрации и не меняется

---

## Что проверять визуально

### В дашборде ученика
- [ ] Плашка с классом под логотипом
- [ ] Класс в профиле
- [ ] Уведомления приходят в реальном времени
- [ ] Задания фильтруются правильно

### В дашборде учителя
- [ ] Информационный блок о системе заданий
- [ ] Выпадающий список классов (1А-11Е)
- [ ] Класс отображается на карточке задания
- [ ] Можно создать несколько заданий

## Отладка

### Если ученик не получает задания:

1. Проверьте консоль браузера на ошибки
2. Проверьте, что класс ученика совпадает с классом задания
3. Проверьте, что событие `new-teacher-task` отправляется
4. Убедитесь, что `studentClass` передан правильно

### Проверка в консоли:

```javascript
// В консоли ученика
console.log('Мой класс:', studentClass);

// В консоли учителя (после создания задания)
window.addEventListener('new-teacher-task', (e) => {
  console.log('Отправлено задание:', e.detail);
});
```
